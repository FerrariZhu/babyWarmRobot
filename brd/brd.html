文档摘要

  • 面向 0–3 岁婴幼儿家庭 的智能穿衣决策工具
  • 核心能力：虚拟衣柜 + 天气感知 + 保暖值算法
  • 输出：科学可执行的每日 OOTD（标准 / 偏暖 / 偏凉）
  • 载体：微信小程序（主）+ Web MVP（当前）

一句话定位：用数据和算法，把爸妈的爱，翻译成刚刚好、不冷不热的穿衣方案。

一、立项背景

从一个家庭的真实焦虑出发

痛点场景

  • 🌡️ 气温 16°C + 风大 → 穿多穿少都难判断
  • 🏃 户外活动多 → 静止与运动保暖需求差异大
  • 🏠 室内室外切换 → 一套衣服难以兼顾
  • 👶 宝宝不会表达 → 家长只能靠「猜」，试错成本高
  • 🧅 理想方案：打底棉衣 + 薄外套 + 防风层（洋葱式穿搭）

立项初衷

  • ❤️ 情感驱动：源于父母对孩子的爱，希望帮助更多家庭
  • 🔍 工具缺失：缺少婴幼儿专属 + 天气 + 衣柜的决策产品
  • 📊 科学育儿：年轻父母接受数据化、可解释的建议
  • 📱 以小程序为主传播渠道，便于宝妈日常使用与分享

三、产品价值

虚拟衣柜 × 天气 × 算法 = 科学 OOTD

核心产品逻辑

精准定位小区级天气
→
温度值目标保暖指数
→
虚拟衣柜单品保暖值
→
分层算法洋葱式穿搭
→
今日 OOTD标准/偏暖/偏凉

温度值 ≈ 各层保暖值之和 → 科学、可解释、可执行的每日穿搭建议

关键术语

术语说明
温度值当日目标保暖指数（0–100），由天气+宝宝档案+场景计算
保暖值单件衣物的保暖贡献，由材质/厚度/类目/版型/克重决定
OOTD从衣柜选出保暖值之和 ≈ 温度值的搭配方案
洋葱式穿搭内层→中层→外层→配件，避免「一件棉袄打天下」

四、三大核心难点

产品成败的关键设计挑战

难点一：低成本录入虚拟衣柜

  • 一个宝宝完整衣柜可达 上百件，存量录入是最大门槛
  • ❌ 爬虫 淘宝反爬极强，不可行
  • ❌ 淘宝 API 仅服务商家 ERP，不对第三方开放
  • ✅ 万邦 API 链接导入材质/价格（约 ¥1000 起）
  • ⚠️ AI 识图 截图 + 分享文案，满足约 80% 信息
  • ✅ 手动录入 极简枚举 + 全屏分类 + 自动算保暖值

难点二：精确计算温度值

  • 城市预报不够精确（上海报小雨，嘉定南翔可能不下）
  • 综合体感温度、湿度、风速、降水概率修正
  • 叠加宝宝月龄、活动量、怕冷/怕热偏好
  • 区分室内 / 外出 / 睡眠三大场景
  • 精确定位到区/街道级获取局部天气

难点三：单件保暖值计算

  • 材质、厚度、类目、尺码、版型、克重均影响结果
  • 宽松棉服保暖弱于修身款 → 量化版型系数
  • 配件（帽/袜/手套）有加分项，非简单叠穿
  • 公式：材质基础分 × 厚度 × 类目 × 版型 + 配件加分 + 克重修正
  • 录入时预览 TOG 等效值，增强用户信任
